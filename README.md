## üé® Design & Motivation

I built this project to simulate a real-world client deliverable. I wanted to move away from "tutorial style" code and build a site that looks trustworthy and expensive that also
tells a bit about me.

I focused heavily on:
1.  **Visual Hierarchy:** Guiding the user's eye to the "Call to Action" buttons.
2.  **Whitespace:** Using negative space to create a clean, professional feel.
3.  **Brand Consistency:** Maintaining a consistent color palette and font usage throughout.

## üí° Lessons Learned

* **CSS Architecture:** I learned how to organize my CSS to keep it maintainable, using variables for colors and fonts.
* **HTML Structure:** I faced a challenge getting the images to load on the page once it was live due to the structuring of my directories which was later fixed.
* **Design:** Simple is sometimes the best option as opposed to over complicating everything.
